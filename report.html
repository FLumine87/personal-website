3. Challenges Faced and Solutions

3.1 [Challenge 1: e.g., Line-Clearing Algorithm Complexity]

[Describe the challenge: Implementing the line-clearing logic that checks all four directions for both shape and colour matches was initially complex, especially handling cases where multiple lines intersect.]

Solution: [Explain how you solved it: Used a directional search approach with positive and negative extensions from the placed block position. Used a Set data structure to eliminate duplicate indices when a block belongs to multiple lines.]

3.2 [Challenge 2: e.g., Multiplayer State Synchronization]

[Describe the challenge: Ensuring all players see the same game state in real-time, especially when players join or leave mid-game.]

Solution: [Explain: Implemented an authoritative server model where all game logic runs on the server. The server broadcasts state updates via Socket.IO after every action. New players receive the full state on connection.]

3.3 [Challenge 3: e.g., Timeout Management with Dynamic Player List]

[Describe the challenge: Managing 60-second timeouts correctly when players join, leave, or their turn is skipped due to timeout.]

Solution: [Explain: Used JavaScript setTimeout with proper cleanup (clearTimeout). When a player times out, the server removes them and recalculates turn order, then sets a new timeout for the next player.]
